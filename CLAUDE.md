# Visual Studio Code Extension "coolors"
Extension for VSCode to explain colors in editor on coolors.co

# Folders
Source Code in /src
Compiled output in /out (gitignored, generated by tsc)

# Technical Stack
- Language: TypeScript 5.8, compiled to CommonJS targeting ES2024
- VSCode engine: ^1.100.0
- Entry point: `src/extension.ts` → compiled to `out/extension.js`
- No runtime dependencies; devDependencies only (eslint, typescript, @types/vscode)
- Build: `npm run compile` (tsc), watch: `npm run watch`

# Implementation Plan
To implement the behavior:

1. **package.json** — register a context menu command:
   - Add command `coolors.explainInCoolors` with title `Explain in coolors.co`
   - Add `menus.editor/context` contribution pointing to that command
   - Set `when` clause to restrict to text editors (optional)

2. **src/extension.ts** — implement the command:
   - Register `coolors.explainInCoolors` with `vscode.commands.registerCommand`
   - In the handler: get the active editor, get cursor position, expand selection to word boundary to extract the hex token (including `#`)
   - Validate the token matches `/^#[0-9a-fA-F]{3,6}$/`
   - Open `https://coolors.co/<hex-without-hash>` as a webview or via `vscode.env.openExternal` — but since spec requires a split tab inside VSCode, use `vscode.window.createWebviewPanel` with `ViewColumn.Beside`
   - Alternatively use a simple `<iframe>` or redirect inside the webview

3. **Webview approach** (split tab inside VSCode):
   - `vscode.window.createWebviewPanel('coolors', 'coolors.co', vscode.ViewColumn.Beside, { enableScripts: false })`
   - Set `webview.html` to an HTML page that embeds `https://coolors.co/<rgb>` in an `<iframe>` or redirects to it

# Hex Color Regex
`/#[0-9a-fA-F]{3,8}/g` — matches 3, 6, or 8-digit hex colors with leading `#`
